# Project 2: Wordsearch
Given a wordsearch puzzle represented as a text document, and a list of words, can you solve the wordsearch puzzle using lists, loops, and file i/o?
You should attempt to implement a solve(puzzle, wordlist) function that returns a JSON-like data structure such as:
{ "CAT": {'position': [[0,2],[1,3],[2,4]] },
	 {'direction': 'downforwards' },
 "...": {'position': [[x,y],[x2,y2],[x3,y4]...] },
	{'direction': 'forwards' OR 'backwards' OR 'upforwards' or 'downforwards' etc.}
}
Where each word in the wordlist is in the output. You should try to get forwards, backwards, up, and down first. Then work on the diagonals.

This one might be kind of hard, feel free to ask for help! Just try not to use anything that would give you the solution outright. You can assume that every word in the wordlist will actually be in the puzzle.
